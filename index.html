Laser Tag App

Previous Next


Overview
This is a capstone project I did in NUS where the aim of the project is to build a laser tag system from scratch. This is a 5-man team project where each member had distinctly different roles as shown above in the first architecture diagram. My role was to design an AR/VR solution that allows the user to visualise their gameplay.

My contributions
I conducted user surveys and came up with the design based on the survey findings. Then I researched on potential technologies that I can use. I prototyped with 3 different technologies:
- Unity + ARFoundation
- Unity + GoogleVR SDK
- Unity + Vuforia10 SDK

After weighing the pros and cons I decided to work with Vuforia10 AR/VR engine with Unity Editor. I developed various user interactions and features on the app to be as user-friendly as possible. Furthermore, to allow for smooth data transfer, we used TCP protocol and I developed the TCP link on Unity.

Some features of the app
- Show player stats as dynamic UI elements
- Show opponent stats as dynamic AR (Augmented Reality) elements
- Animations and triggers according to player and opponent actions such as:
     -> Shooting
     -> Shielding
     -> Throwing grenades
     -> Reloading
- Link to backend using TCP client-server protocol

Overcoming challenges
One of the biggest challenges is to integrate the different technologies of the 5 group member's contributions together. We overcame this challenge by learning about each other's parts and understanding how they work helped with integration. One takeaway is that when working in a team, it is important to be open to learn technologies, concepts and ideas that are out of scope of your own work to better understand the bigger picture which would help to further improve the entire system as a whole.

Meeting Room Controller

Previous Next


This is a project I did as a GovTech intern. This is POC (Proof-of-concept) project where I developed a mobile and tablet flutter app to remotely control meeting room equipment. I also developed and integrated the API into the flutter app such that the app fetches the data from MongoDB database and EnOS server when the user signs in. The authentication happens via a QR code. Through this authentication, room specific data (e.g. room name, input and output sources, last saved preferences) will be fetched. I also set up the infrastructure that links the various components via API and MQTT (shown on the last slide). The prototype is complete and is fully functional. The user is able to control the equipment from both the mobile and the tablet.

Some features of the app:
- Drag and drop the input source to connect to the preferred output source
- Let users customise the design for specific rooms which will be automatically fetched during sign-in
- Adjust a single source volume using a slider and mute/unmute function
- Scroll through a list of input and output sources available
- Turn the devices on and off with the power button

Device Management App

Previous Next


This is a project I did as a GovTech intern. The idea behind this project is to create a device management app for users to navigate across various locations (building assets) and view information about the devices in the location. The tricky part of this project is to build the data model such that creating/modifying the hierarchy of the building assets can be easily maintained and expanded for future use cases. Initially, the data model was created in MongoDB as JSON. However, fetching data from a linear JSON structure was too inefficient. So in efforts to improve time and memory efficiency, I explored the following technologies and implemented them:
- Web ontologies, BrickSchema: This is to create a consistent building assets' based data model
- Neo4j and Neosemantics: This is to import/export the data model and store it in a graph database
- GraphQL API and Neo4j/GraphQL JS library: This is to create the API to expose the data for the app

The app is fully functioning and I added it as a service to a larger smart facility management app. Some features of the app are as follows:
- Navigate between parent and child building assets/locations
- View the child assets as a list
- Search any asset and directly navigate to that asset
- View detailed device info
- Navigate back to home page with one click

EMeter App

Previous Next


This is a project I did as a GovTech intern. The idea behind this project is to create a data visualisation app for users to view and monitor eMeter data. (building assets) and view information about the devices in the location. There are 2 kinds of view perspective, viewing individual emeters and viewing all emeters. There are 3 different kinds of displays (bar chart, pie chart and line chart) representing different data. The challenge behind this app is to fetch large amounts of timeseries data, organise the data in a usable structure and fit them to the charts. One of the major aims is to optimise the time and space efficiency for a better user experience. The app is fully functioning and I added it as a service to a larger smart facility management app.

Some features of the app are as follows:
- View energy usage by month in a bar chart. This usage can be toggled by year
- View energy usage in comparison with other emeters in a pie chart. This usage can be toggled by year.
- The pie chart is interactive such that it highlights the selected pie section
- View usage of various other parameters of an individual eMeter in a line chart. This timeseries data can be toggled by time period (day, week, month).
- View usage of any parameter of all eMeters in a line chart. This timeseries data can be toggled by time period and paramter.
- The line chart is interactive such that only the selected line charts are displayed/changed dynamically
